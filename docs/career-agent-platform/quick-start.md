# Ragent Career Agent Platform 快速开始

## 项目一句话
这是 Ragent 里的职业闭环，不是新系统。Ragent 继续作为唯一运行基座，Career 域只承接简历、JD、优化、面试和报告这些业务状态。

## 演示顺序
1. 进入 `/career`，确认统一壳、登录和模型路由都沿用 Ragent。
2. 上传简历并完成解析，展示版本号、结构化字段和 Trace。
3. 创建目标 JD 并做对齐，展示评分、证据和缺口。
4. 生成优化建议，展示裁判-执行者循环、`0.8` 质量门禁和可追踪的修订记录。
5. 开始文字面试，展示题目、作答、评分、追问、手动重试和会话恢复。
6. 生成面试报告，展示总分、雷达维度、风险和复盘建议。
7. 打开 `/admin/career/tasks`，查看任务状态、失败原因和 Trace 链接。

## 讲解重点
- Ragent 是唯一运行基座：统一认证、模型路由、存储、Trace 和前端壳都不换。
- AI-Meeting 的面试运行时治理已经吸收进来：题目状态机、单题幂等、补偿 worker、手动重试、会话恢复和 Single-flight 都服务于文字面试闭环。
- JobNavigator 的裁判-执行者优化和渲染/检索亮点已经吸收进来：质量门禁、HyDE/Rerank、进度事件和多格式导出都成为 Career 域能力，不再是独立子系统。
- Career Skill Pack 是项目知识资产：Rubric、Prompt、状态机、排障和验收口径都按版本沉淀，供 Agent 直接消费。

## 可直接复述的话术
“这个项目不是把两个系统拼起来，而是把 Ragent 扩成唯一运行基座，把 AI-Meeting 的面试治理和 JobNavigator 的优化、检索、渲染能力吸收到 Career 域。演示时先跑简历、JD、优化，再跑文字面试和报告，最后用任务和 Trace 证明每一步都能追溯。”

## 演示时不要这么讲
- 不是再做一套 AI-Meeting。
- 不是再做一套 JobNavigator。
- 不是新增第二套运行时、第二套模型路由或第二套 Trace。
- 不是把自由文本当成最终结果，核心结论都要回到版本、证据和 Trace。
